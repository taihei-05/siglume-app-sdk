## Coverage legend

- `typed`: the SDK has a dedicated high-level wrapper for the operation semantics.
- `generic-only`: no dedicated wrapper exists yet; the current escape hatch is `SiglumeClient.execute_owner_operation()` plus `list_operations()` / `get_operation_metadata()` when the public owner-operation surface exposes the key.
- `not exposed`: the operation exists in the registry, but the public SDK intentionally does not surface it yet. Today this bucket is mostly `account.avatar.upload` plus `admin.*` operations that remain out of scope for public SDK consumers.

## S2 candidates

Priority order for the next typed-wrapper slices after PR-S2b / PR-S2d / PR-S2e:

1. `wallet.balance.get` / `wallet.accounts.list`
Reason: the wallet namespace is now the cleanest remaining owner dashboard slice that is still read-heavy and independent from account/profile churn.
2. `market.approvals.list` / `market.approvals.get` / `market.approvals.approve`
Reason: this completes the owner-side negotiation loop that `market.needs.*` and `market.proposals.*` now cover.
3. `partner.keys.revoke` / `partner.billing.checkout`
Reason: partner reads are typed already, so the next highest-value follow-up is finishing the partner mutation path without opening the broader wallet or ads lifecycle.

Namespace ordering rationale: with `market.*`, `works.*`, `installed_tools.*`, and the key `partner.*` / `ads.*` reads typed, the safest remaining slices are wallet reads first, then market approval actions, then narrowly-scoped operator mutations.
